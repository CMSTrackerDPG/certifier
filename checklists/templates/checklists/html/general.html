{# extends either base.html or modal-basel.html #} {% extends checklist_base_template_name %} {% block title %}TkDQM Doc: General Help{% endblock %} {% block checklist_block_id %}general-id{% endblock %} {% block checklist_headline %}General Checklist{% endblock %} {% block checklist_content %} These are basic checks and actions to be performed once the certification of a run start. Dediced checks for Tracker Maps, Pixel, SiStrip and Tracking are presented in other check list section (accessible frop the "Add a Run" form. Before starting the run inspection, please check the correct dataset in the DQM Offline GUI:
  • In pp collision the dataset to use is StreamExpress, for prompt reco runs ZeroBias is to be used,
  • In case of cosmic run, you have to check StreamExpressCosmics and Cosmics for the prompt reco
  • For heavy ions data taking the dataset to check are StreamHiExpress and MinimumBias1

General Check List

  • Check is the sample is finished (Workspace->Info on the GUI). If more that 70% of the LS are processes, produce the TrackerMaps (you may need to use the "force" option if the run is not 100% processed).
  • Check the number if LS and the number of events. Be aware that, in case of low stat, some quality test might not be robust and coudl give bad resutls while the tracker is fine.
  • Check the amount of LS for which the HV is on.
  • Check the trigger rates (go to the RunSummary), make sure the trigger rate is non-zero.
  • Choose your reference run (and set it in the GUI to be overlayed to all curves. Using RunSummary, select a reference runs which has similar beam conditions than the run under certification.
  • One the GUI, go to the "Summary" workspace, and check the summaries for SiStrip and Tracking. If SiStrip summary shows fractions > 95%, you already have an indication on the data quality
  • Check quickly look at the "Tracking" workspace. Make sure you do see tracks.
  • Create the tracker maps, following the instruction given on link.
